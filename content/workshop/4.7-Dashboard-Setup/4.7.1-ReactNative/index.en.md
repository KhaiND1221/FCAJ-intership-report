# 4.7.1 Expo Setup

## Project structure

The NutriTrack frontend lives in `frontend/` alongside the Amplify backend at `backend/`. If you are building from scratch, initialize with:

```bash
npx create-expo-app@latest frontend --template blank-typescript
cd frontend
```

The repo already has `frontend/` — clone it and install:

```bash
cd frontend
npm install   # --legacy-peer-deps enforced by .npmrc
```

## Amplify configuration

`amplify_outputs.json` is generated by the Amplify pipeline and contains all endpoint URLs, region config, and API identifiers for the active environment. Never commit manual edits — it is overwritten on every `npx ampx sandbox` run or CI deploy.

Initialize Amplify once in `src/lib/amplify.ts`:

```typescript
import { Amplify } from 'aws-amplify';
import outputs from '../../amplify_outputs.json';

Amplify.configure(outputs);
```

Import this as a side-effect in the root layout so it runs before any component:

```typescript
// app/_layout.tsx
import '@/src/lib/amplify';  // must be first
import { GestureHandlerRootView } from 'react-native-gesture-handler';
import { LanguageProvider } from '@/src/i18n/LanguageProvider';
// ...
```

To regenerate `amplify_outputs.json` after backend changes:

```bash
cd backend
npx ampx generate outputs --outputs-out-dir ../frontend
```

## Expo Router file-based routing

Every file in `app/` is a route. The structure:

```text
app/
  _layout.tsx          → root Stack navigator + auth guard
  (tabs)/
    _layout.tsx        → Tab navigator (home, kitchen, battle, ai-coach, progress, add)
    home.tsx           → /
    kitchen.tsx        → /kitchen
    battle.tsx         → /battle
    ai-coach.tsx       → /ai-coach
    progress.tsx       → /progress
    add.tsx            → /add
  welcome.tsx          → /welcome
  login.tsx            → /login
  signup.tsx           → /signup
  verify-otp.tsx       → /verify-otp
```

Expo Router infers the navigator type from the folder name: `(tabs)/` uses a Tab navigator; the root `_layout.tsx` wraps everything in a Stack.

## Guest Mode — Cognito Unauthenticated Identities

NutriTrack allows unauthenticated users to scan food barcodes before signing in. This requires **Guest Access (Unauthenticated Identities)** to be enabled on the Cognito Identity Pool — without it the app cannot make any AWS API calls for guests and the barcode scanner will fail silently.

Enable it in `backend/amplify/auth/resource.ts`:

```typescript
import { defineAuth } from '@aws-amplify/backend';

export const auth = defineAuth({
  loginWith: {
    email: true,
    externalProviders: {
      google: { ... },
      callbackUrls: [...],
      logoutUrls: [...],
    },
  },
  // Required for guest barcode scan before sign-in
  guestAccess: 'enabled',
});
```

Then attach a scoped IAM policy to the **unauthenticated role** in `backend.ts` so guests can only invoke the specific Bedrock/AppSync actions needed for food lookup — they must not be able to write food logs or access other users' data:

```typescript
const { unauthenticatedUserIamRole } = backend.auth.resources;

unauthenticatedUserIamRole.addToPolicy(new iam.PolicyStatement({
  effect: iam.Effect.ALLOW,
  actions: ['appsync:GraphQL'],
  resources: [`${backend.data.resources.graphqlApi.arn}/types/Query/*`],
}));
```

> **Without this step**, guests receive an `UnauthorizedException` the moment they open the barcode scanner and the feature is completely non-functional.

## Auth guard in root layout

The root `_layout.tsx` checks authentication state on mount and redirects unauthenticated users to `/welcome`. The real implementation (~390 lines) adds `Hub.listen('auth')` for OAuth redirect handling, biometric prompts, and `useRef` guards. The core structure:

```typescript
// app/_layout.tsx (simplified)
import { useEffect } from 'react';
import { router, Slot } from 'expo-router';
import { getCurrentUser } from 'aws-amplify/auth';
import { LanguageProvider } from '@/src/i18n/LanguageProvider';
import { GestureHandlerRootView } from 'react-native-gesture-handler';

export default function RootLayout() {
  useEffect(() => {
    getCurrentUser()
      .then(() => { /* authenticated — stay */ })
      .catch(() => router.replace('/welcome'));
  }, []);

  return (
    <GestureHandlerRootView style={{ flex: 1 }}>
      <LanguageProvider>
        <Slot />
      </LanguageProvider>
    </GestureHandlerRootView>
  );
}
```

The full layout also subscribes to `Hub.listen('auth', ...)` so that when `signInWithRedirect` completes (Google OAuth), the auth state updates immediately and the router navigates to `(tabs)/home` without requiring another `getCurrentUser` poll.

The `LanguageProvider` wraps the entire app so all screens can access `useAppLanguage()` for Vietnamese/English switching.

## Biometric authentication

After sign-in, on every app resume the security layer prompts biometric authentication (fingerprint or Face ID). This uses `expo-local-authentication`:

```typescript
import * as LocalAuthentication from 'expo-local-authentication';

const result = await LocalAuthentication.authenticateAsync({
  promptMessage: 'Xác thực để tiếp tục',
  fallbackLabel: 'Dùng mật khẩu',
});
if (!result.success) {
  // lock the app or sign out
}
```

**Important**: `expo-local-authentication` requires a native build. It does not work in Expo Go. Use `npm run android` or `npm run ios` (or an EAS development build) to test biometric flows.

## Running the app

```bash
# Expo Go (fastest, but no biometric / native modules)
npm start          # scan QR in Expo Go

# Full native (requires Android Studio / Xcode)
npm run android
npm run ios

# Web browser
npm run web

# Production web export
npm run build      # outputs to frontend/dist/
```

## Path alias

`@/` maps to `frontend/` in `tsconfig.json`:

```json
{
  "compilerOptions": {
    "baseUrl": ".",
    "paths": { "@/*": ["./*"] }
  }
}
```

Use `@/src/store/authStore` instead of `../../src/store/authStore` throughout the codebase.

## Troubleshooting

| Symptom | Cause | Fix |
| --- | --- | --- |
| `npm install` fails with peer-dep errors | Missing `.npmrc` | Add `legacy-peer-deps=true` to `frontend/.npmrc` |
| Metro can't resolve `amplify_outputs.json` | File missing | Run `npx ampx generate outputs --outputs-out-dir ../frontend` from `backend/` |
| Biometric prompt not showing | Running in Expo Go | Use `npm run android` / `npm run ios` for a dev build |
| `Cannot find module 'aws-amplify'` | Install not done | `cd frontend && npm install` |
| Screen flash between auth and tab navigator | Race condition on `isAuthenticated` init | Add a loading state to `authStore`; show a splash until state is hydrated |
